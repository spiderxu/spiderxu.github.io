MySQL索引基本介绍

MySQL
数据库 MySQL
发布日期:   2024-03-31
文章字数:   1.2k
阅读时长:   4 分

什么是索引

索引，说到底是一种帮助MySQL提交查询效率的数据结构。运用现实生活的中的案例，可以理解为书的目录。比如说我们要看一本书的某个章节在第几页，我们不会一页一页的去翻，而是先查目录，查到章节在第几页后，就可以直接跳到那一页。比一页一页的翻效率高太多了。

索引的优点
大大加快数据的查询效率

索引的缺点
1、维护索引需要耗费数据库的资源
2、索引需要占用磁盘空间
3、当对表的数据进行修改的时候，因为要维护索引，速度会受到影响

索引分类

主键索引

设定为主键后数据库会自动建立索引，innodb为聚簇索引。主键索引索引列的值不能为空

单值索引（普通索引，单列索引）
即一个索引只包含单个列，一个表可以有多个单列索引

唯一索引
索引列的值必须唯一，但允许有空值。空值只能存在一个

复合索引
一个索引包含多个列

Full Text全文索引 （5.7版本之前，只能由MYISAM存储引擎）
全文索引的类型为FullText，在定义索引的列上支持值的全文查找，允许在这些索引列中插入重复值和空值。全文索引可以在CHAR,VARCHAR、TEXT列类型上创建，MYSQL只有MYISAM存储引擎支持全文索引。

索引的基本类型

1、主键索引，我们指定主键的时候会自动创建，无需我们操心

--创建一个t_user表，指定主键
create table t_user(id varchar(20) primary key,name varchar(20));

--查看t_user表的索引
show index from t_user;

2、单值索引
两种创建方式：建表时创建，建表之后创建

--建表之后创建
create index name_index on t_user(name);

--建表的同时创建索引，建表时不能指定索引名称，默认就是字段名
create table t_user(id varchar(20) primary key,name varchar(20),key(name));

3、唯一索引

--建表之后创建
create unique index name_index on t_user(name);

--建表的同时创建索引，建表时不能指定索引名称，默认就是字段名
create table t_user(id varchar(20) primary key,name varchar(20),unique(name));

4、复合索引

create  index name_index on t_user(name,age);

--建表的同时创建索引，建表时不能指定索引名称，默认就是字段名
create table t_user3(id varchar(20) primary key,name varchar(20),age int,key(name,age));

最左前缀原则

索引的底层原理

create table t_emp(id int primary key,name varchar(20),age int);

1、插入id无序
2、查询得到的结果有序

3、主键索引进行了排序

聚簇索引与非聚簇索引

聚簇索引：将数据存储与索引放到了一块，
非聚簇缩影：将数据与索引分开存放

聚簇索引需要注意什么？
当使用主键为聚簇索引的时候，主键最好不要使用UUID，应为UUID的值太过离散，不适合排序。并且新增记录的UUID可能会插在索引中间树的位置上导致索引树调整复杂度变大，消耗更多的时间和资源
建议使用int类型自增，方便排序并且花在索引树的末尾增加主键值，对索引树的结构影响最小，而且主键占用空间越大，辅助索引中保存的主键值也会跟着变大，占用存储空间。也会影响到IO操作读取到的值。

为什么主键通常建议使用自增id
聚簇索引的数据的物理存放顺序与索引顺序是一致的，也就是说只要索引是相邻的，那么对应的数据也一定是相邻的存放在磁盘上，如果主键不是自增id,那么将会不断调整数据的物理地址，分裂。

什么情况下无法使用索引呢
1、查询语句中使用Like语句
在查询的过程中使用Like关键字进行查询的时候，如果匹配字符串的第一个字符为%，那么索引便不会被使用
2、查询语句中使用多列索引
多列索引是在表的多个字段上创建了一个索引，只有再查询条件上使用了第一个字段，索引才会被使用
查询语句中使用了or关键字
查询语句中只有or关键字，如果or前后两个条件的列都是索引，那么查询中将使用索引，如果or前后有一个不是索引，那么查询中将不使用索引
